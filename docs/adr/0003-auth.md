# ADR 0003 — Kimlik Dogrulama ve Yetkilendirme

- **Durum:** Kabul edildi
- **Tarih:** 2026-06-04
- **Ilgili:** [`auth/`](../../apps/api/src/auth), ADR 0001 (JWT karari)

## Baglam

Admin paneli (icerik yonetimi) ve public API ayni NestJS uygulamasinda. ADR 0001'de JWT
secildi; bu ADR uygulama detaylarini kayda gecirir.

## Kararlar

1. **Access + Refresh token.** Access kisa omurlu (15 dk), refresh uzun (7 gun).
   - *Neden:* Calinan access token'in penceresi kucuk; oturum refresh ile sessiz uzar.
2. **Tasima: httpOnly cookie + `Bearer` fallback.** Tokenlar httpOnly cookie'de (SameSite=Lax,
   uretimde Secure); guard ayrica `Authorization: Bearer`'i da kabul eder.
   - *Neden:* Cookie => XSS'e karsi token JS'ten okunamaz (admin guvenligi). Bearer fallback =>
     Swagger/test ve programatik istemci kolayligi.
   - *Alternatif:* localStorage + Bearer (XSS riski), sunucu oturumu (stateful). **Reddedildi.**
3. **Refresh rotation + DB'de hash.** Her refresh'te eski token revoke, yenisi uretilir; token
   DB'de **sha256 hash** olarak (`RefreshToken`) tutulur (duz deger degil).
   - *Neden:* Revocation/iptal mumkun (logout, calinma). DB-tabanli => Redis'e bagimli degil,
     kalici ve test edilebilir. sha256 deterministik => indeksten hizli arama.
   - *Alternatif:* Redis denylist (ADR 0001'de anilmisti) — kalicilik/test icin DB tercih edildi.
4. **RBAC.** Roller `ADMIN` / `EDITOR`; `JwtAuthGuard` (kimlik) + `RolesGuard` + `@Roles()`
   (yetki). `@CurrentUser()` ile kullanici enjekte edilir.

## Sonuclar / dikkat

- **CSRF:** Cookie tabanli mutasyonlar icin SameSite=Lax ilk savunma; uretimde ayrica CSRF
  token eklenmeli (Faz 9 notu).
- Refresh cookie path'i `/api/auth` ile sinirli (yuzey azaltma).
- Secret'lar `.env` (JWT_ACCESS_SECRET / JWT_REFRESH_SECRET); uretimde rotasyon onerilir.

## Dogrulandi (canli)

login → ADMIN token; `/auth/me` (Bearer) calisiyor; tokensiz istek **401**; rate limit aktif.
